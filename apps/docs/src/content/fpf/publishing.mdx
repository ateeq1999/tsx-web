# Publishing a package

This guide walks through the full workflow for creating and publishing an FPF package to
the registry — from scaffolding to a live install command.

## Prerequisites

- tsx CLI installed (`tsx --version`).
- An account on the registry with an API key (generate one in the dashboard under *API Keys*).

## Step 1 — Scaffold the package

```bash
mkdir my-pattern && cd my-pattern
tsx framework init
```

This creates `manifest.json`, `generators/`, and `templates/`.

## Step 2 — Edit the manifest

Open `manifest.json` and fill in the required fields:

```json
{
  "id": "my-pattern",
  "name": "My Pattern",
  "description": "A reusable pattern that adds X to TanStack Start projects.",
  "version": "1.0.0",
  "tsx_min": "0.4.0",
  "lang": "typescript",
  "provides": ["my-capability"],
  "generators": [
    {
      "id": "setup",
      "command": "generators/setup.rhai",
      "description": "Scaffold the integration.",
      "schema": {},
      "output_paths": ["src/lib/my-pattern.ts"]
    }
  ]
}
```

## Step 3 — Write generators and templates

Edit `generators/setup.rhai` to implement the scaffolding logic.
Use `write_template("templates/my-file.ts.hbs", ctx)` to render a Handlebars
template and write it to an output path.

```javascript
// generators/setup.rhai
let ctx = #{
  project_name: input.project_name,
};
write_template("templates/my-pattern.ts.hbs", ctx);
```

Then edit `templates/my-pattern.ts.hbs`:

```handlebars
// Auto-generated by tsx install {{id}}
export const myPattern = "{{project_name}}"
```

## Step 4 — Validate

```bash
tsx framework validate
```

Fix any reported errors before proceeding. Common issues:

- Missing required fields in `manifest.json`.
- Generator script path not found.
- Invalid semver in `version` or `tsx_min`.

## Step 5 — Preview

Run the generator locally to confirm the output looks correct:

```bash
tsx framework preview --generator setup
```

## Step 6 — Publish

```bash
tsx framework publish --api-key sk_live_YOUR_KEY_HERE
```

tsx will bundle the package into a `.tar.gz` archive and upload it to the registry.
On success, the install command is printed:

```bash
tsx install my-pattern
```

## Publish to a self-hosted registry

```bash
tsx framework publish \
  --registry https://registry.my-org.com \
  --api-key sk_live_...
```

## Bumping versions

Update the `version` field in `manifest.json` before each publish.
The registry rejects duplicate version uploads. Follow semver:

- **Patch** (`1.0.1`) — Bugfixes, no breaking changes.
- **Minor** (`1.1.0`) — New generators or fields, backwards compatible.
- **Major** (`2.0.0`) — Breaking changes to generators or manifest structure.
