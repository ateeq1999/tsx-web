# Official Packages

Official `@tsx-pkg/*` packages are maintained by the tsx project and published to the
[public registry](https://registry.tsx.dev). They follow the [FPF format](/docs/fpf),
are version-pinned via `stack.json`, and can be installed with a single command.

Each package declares a `provides[]` list of capability tokens. Other packages can declare
`integrates_with` on these tokens to automatically inject code through the slot system тАФ
no manual wiring required.

### `@tsx-pkg/with-auth`

Full [Better Auth](https://better-auth.com) integration for TanStack Start.

**What gets scaffolded:**

```
src/
тФЬтФАтФА lib/
тФВ   тФЬтФАтФА auth.ts            # Server-side better-auth instance + Drizzle adapter
тФВ   тФФтФАтФА auth-client.ts     # Client-side createAuthClient()
тФЬтФАтФА middleware/
тФВ   тФФтФАтФА auth-guard.ts      # requireAuth() тАФ redirects unauthenticated users
тФФтФАтФА routes/
    тФЬтФАтФА auth/
    тФВ   тФЬтФАтФА login.tsx       # Login form (email + password)
    тФВ   тФЬтФАтФА register.tsx    # Sign-up form with email verification
    тФВ   тФЬтФАтФА forgot-password.tsx
    тФВ   тФФтФАтФА reset-password.tsx
    тФФтФАтФА _protected/
        тФФтФАтФА dashboard.tsx   # Example protected route
```

**Provides:** `auth`, `session`, `protected-routes`

**Integrates with:** `@tsx-pkg/basic-crud` (injects session guard into CRUD route `beforeLoad`)

### `@tsx-pkg/full-saas`

Batteries-included SaaS starter. Combines auth, teams/organizations, role-based access

**What gets scaffolded:**

**Provides:** `auth`, `billing`, `teams`, `rbac`, `dashboard`

## Capability token conventions

Use these standard tokens in your `provides[]` where applicable to maximise
interoperability with other packages:

| Token | Meaning |
|-------|---------|
| `auth` | Provides authentication (sign-in/sign-out) |
| `session` | Provides server-side session storage |
| `protected-routes` | Provides route-level auth guard middleware |
| `database` | Provides a database client/connection |
| `migrations` | Provides schema migration tooling |
| `schema` | Provides a Drizzle (or other ORM) schema |
| `crud` | Provides create/read/update/delete operations |
| `server-functions` | Provides TanStack Start server functions |
| `optimistic-ui` | Provides optimistic mutation UI patterns |
| `ui-components` | Provides a UI component library |
| `tailwind` | Provides Tailwind CSS setup |
| `design-system` | Provides a theme/design token system |
| `router` | Provides router configuration |
| `query-client` | Provides a TanStack Query client |
| `env-validation` | Provides Zod-validated env schema |
| `rbac` | Provides role-based access control |
| `billing` | Provides billing/subscription infrastructure |
| `teams` | Provides multi-tenant org/team model |
| `dashboard` | Provides an admin/user dashboard shell |
| `i18n` | Provides internationalisation support |
| `analytics` | Provides analytics tracking |
| `storage` | Provides file/object storage |
| `email` | Provides transactional email sending |
