# Getting Started

tsx is a CLI for installing and publishing reusable full-stack code patterns into [TanStack Start](https://tanstack.com/start) projects.
Think of it like shadcn/ui, but for complete patterns — auth flows, CRUD with optimistic UI,
Drizzle ORM setups, role-based access, and more.

## How it works

A tsx **package** (called a _Framework Package_) is a versioned bundle of:

- **Generator scripts** — Rhai or shell scripts that scaffold files into your project structure.
- **Templates** — Handlebars templates for the generated source files.
- **A manifest** — `manifest.json` describing what the package provides and how it slots into others.

When you run `tsx install with-auth`, tsx:

1. Downloads the package tarball from the registry.
2. Copies generators, templates, and the manifest into `.tsx/packages/with-auth/`.
3. Records the pinned version in `.tsx/stack.json`.
4. Automatically wires any slot injections into other installed packages.

No runtime dependency is added to your project — tsx writes source files directly, not wrappers.

- A [TanStack Start](https://tanstack.com/start) project (or any compatible Vite + React setup).

## Quick start

### 1. Install the CLI

```bash
cargo install tsx-forge
```

Or download a pre-built binary from the [GitHub Releases page](https://github.com/ateeq1999/tsx/releases).

### 2. Navigate to your project

```bash
cd my-tanstack-app
```

### 3. Install your first package

```bash
tsx install with-auth
```

tsx scaffolds a full Better Auth integration directly into your project:

```
src/
├── lib/
│   ├── auth.ts            # Server-side auth config
│   └── auth-client.ts     # Client-side auth instance
└── middleware/
    └── auth-guard.ts      # requireAuth() helper
```

No manual wiring needed — the files are ready to use.

## Browse and discover packages

Search the registry from the terminal:

```bash
tsx search crud
tsx search --lang typescript
tsx search auth --size 10
```

Or browse visually at [registry.tsx.dev](https://registry.tsx.dev).

## Install multiple packages — automatic integration

Packages integrate with each other automatically through the _slot system_.
Install auth and CRUD together:

```bash
tsx install with-auth
tsx install tanstack-crud
```

Because `with-auth` declares `integrates_with.tanstack-crud` in its manifest, tsx
automatically injects the auth guard into the CRUD route templates — no manual edits required.

---

## Reproduce a setup across machines

After installing packages, `.tsx/stack.json` records all installed versions:

```json
{
  "version": 1,
  "packages": {
    "with-auth": "1.3.0",
    "tanstack-crud": "2.1.0"
  }
}
```

Commit this file. Anyone cloning the project can restore the exact same setup:

```bash
tsx stack apply --install
```
