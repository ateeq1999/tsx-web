# First-party Packages

Official `@tsx-pkg/*` packages are maintained by the tsx project and published to the
[public registry](https://registry.tsx.dev). They follow the [FPF format](/docs/fpf)
and can be installed with one command.

Each package declares a `provides[]` list of capability tokens. Other packages can declare
`integrates_with` dependencies on these tokens, enabling automatic slot injection.

## Publishing your own

Anyone can publish packages to the registry. See the [Publishing guide](/docs/fpf/publishing) to get started.
Community packages appear alongside first-party packages in search results.

## Providing capability tokens

The `provides[]` field in `manifest.json` lists abstract capability tokens
your package fulfills. For example, a package that sets up authentication should list `"auth"`.
Other packages can then declare `integrates_with` to inject code into your package's slots.

Standardised token names ensure interoperability — use the tokens from existing official packages
where possible, or introduce new ones with clear documentation.
